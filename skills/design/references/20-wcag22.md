# 20 — Accessibility: the WCAG 2.2 AA checks that bite

Accessibility is not a phase. It is a constraint on every interactive element. This module
is the **mechanical floor**: the checks that actually fail real audits. Automated tools
(axe-core) catch ~57% of issues by volume — the rest need the manual checks below.

## Why agents fail here by default

They ship div-buttons, skip labels, forget focus states, rely on color alone, and never
tab through the UI. Every one of these is caught by a rule below.

## The non-negotiable floor (WCAG 2.2 Level AA)

1. **Semantic HTML first.** A button is `<button>`, a link is `<a href>`. Never a `<div onClick>`.
   Native elements bring keyboard + role + focus for free. Reach for ARIA only when no
   native element fits (see `21-aria-apg.md`).
2. **Every input has a programmatic label** (`<label for>`, `aria-label`, or `aria-labelledby`).
   Placeholder is not a label.
3. **Contrast:** text ≥ 4.5:1 (≥3:1 for large ≥24px or ≥18.66px bold); UI components and
   meaningful graphics ≥ 3:1. Prefer APCA for judgment, WCAG 2 as the gate (`11-color.md`).
4. **Never convey information by color alone** (1.4.1). Error states need an icon/text, not
   just red. Links in body text need a non-color affordance.
5. **Visible focus indicator** on every focusable element (2.4.7) with ≥3:1 contrast against
   adjacent colors and a minimum area (2.4.11 Focus Appearance, new in 2.2). Use box-shadow
   rings, not `outline` (`14-shape-elevation.md`). Never `outline: none` without a replacement.
6. **Full keyboard operability** (2.1.1) with no keyboard trap (2.1.2). Everything clickable
   is reachable and operable by keyboard. Logical tab order (`22-keyboard-focus.md`).
7. **Target size ≥ 24×24 CSS px** (2.5.8, new in 2.2); expand the hit area if the visual is
   smaller. Mobile touch targets ≥ 44×44. (Vercel/Apple HIG converge here.)
8. **Text resizes to 200%** without loss of content (1.4.4), and **reflows at 320px width**
   (1.4.10) with no horizontal scroll. Use rem/relative units, never fixed px for text.
9. **`prefers-reduced-motion` respected** (`23-motion-a11y.md`). No animation that can't be
   disabled; nothing flashing >3×/sec (2.3.1).
10. **Page has one `<h1>`, headings nest without skipping** (1.3.1), landmarks present
    (`<header><nav><main><footer>`), `<html lang>` set.
11. **Images have alt** (1.1.1): descriptive for meaningful, `alt=""` for decorative.
12. **Focus not obscured** (2.4.11, new in 2.2): sticky headers/footers must not cover the
    focused element. **Dragging has a single-pointer alternative** (2.5.7, new in 2.2).
13. **Accessible authentication** (3.3.8, new in 2.2): no cognitive-function test (e.g.
    solving a puzzle, transcribing) without an alternative. Allow paste into password fields.

## New in WCAG 2.2 (the ones people miss)

`2.4.11 Focus Not Obscured` · `2.4.13 Focus Appearance` · `2.5.7 Dragging Movements` ·
`2.5.8 Target Size (Minimum)` · `3.2.6 Consistent Help` · `3.3.7 Redundant Entry` ·
`3.3.8 Accessible Authentication`. (`4.1.1 Parsing` was **removed** in 2.2 — don't cite it.)

## Manual pass (do this every time, tools can't)

- [ ] Tab through the entire flow: order logical, focus always visible, no trap
- [ ] Operate every control with keyboard only (Enter/Space/arrows/Esc as appropriate)
- [ ] Zoom to 200% and 400% (320px reflow) — nothing clipped or overlapping
- [ ] Turn on `prefers-reduced-motion` — essential motion still works, decorative motion stops
- [ ] Check one path with a screen reader (VoiceOver/NVDA): names, roles, states announced
- [ ] Unplug color: is every state still distinguishable?
