# 71 — CSS in 2026 (Tailwind v4 + modern platform)

Use the platform. A lot of what used to need JS or hacks is now native CSS with wide support.
Prefer native; reach for JS only when CSS can't express it.

## Tailwind v4

- **CSS-first config:** `@theme` in CSS, not `tailwind.config.js`. Tokens are CSS variables;
  colors are **OKLCH** by default (`11`). `@import "tailwindcss"`.
- **Container queries built in:** `@container` + `@sm/@md/...` container variants — use for
  reusable components (`13`).
- Every shadcn primitive carries a `data-slot`; style via data attributes, not deep selectors.
- Lint/order with `eslint-plugin-better-tailwindcss` + `prettier-plugin-tailwindcss` (`82`).
- Use `gap`, not `space-x/space-y` (`13`). Use logical properties for i18n (`61`).

## Native platform features to prefer (2026, broadly supported)

| Feature | Use it for | Note |
|---|---|---|
| **Container queries** (`@container`) | components that respond to their container | replaces most component-level breakpoints |
| **`:has()`** | parent/prev-sibling styling, form state | huge — style a card based on its contents |
| **Cascade layers** (`@layer`) | predictable specificity (reset/base/components/utilities) | tames override wars |
| **`@property`** | typed custom props, animatable gradients/angles | enables animating variables |
| **Anchor positioning** | tether popovers/tooltips to triggers in CSS | pairs with Popover API; check support + fallback |
| **Popover API** (`popover` attr) | native top-layer popovers/menus | native light-dismiss, no JS state |
| **`<dialog>`** | modals with native focus trap + top layer | use over hand-rolled modals (`37`) |
| **View Transitions** | animated state/page changes | native continuity (`41`) |
| **Scroll-driven animations** | scroll-linked effects on compositor | cheaper than JS parallax (`41`) |
| **`@starting-style` / `allow-discrete`** | animate enter from `display:none` / top layer | pure-CSS popover/dialog transitions |
| **`color-mix()`** / relative color | derive tints/shades from a token | tint neutrals toward hue (`11`) |
| **`text-wrap: balance/pretty`** | headline/paragraph wrapping quality | cheap typographic polish (`12`) |
| **`clamp()` / fluid type** | responsive type/space without breakpoints | mind a11y zoom (`20`) |
| **Subgrid** | align nested grids to a parent grid | complex layouts |
| **`field-sizing: content`** | auto-growing textareas/inputs | forms (`33`) |

## Rules (enforceable)

1. **Native before library.** `<dialog>`/Popover/anchor before a JS modal/popover lib; scroll-
   driven CSS before JS scroll listeners; View Transitions before a page-transition lib.
2. **Always provide fallbacks** for newer features (`@supports`) — anchor positioning and some
   others still need graceful degradation. Don't ship a hard dependency on bleeding-edge CSS
   without a fallback path.
3. **Cascade layers for any non-trivial CSS** so specificity is predictable and overrides are
   intentional.
4. **Container queries for components, viewport queries for page layout** (`13`).
5. **Logical properties everywhere** for i18n/RTL (`61`).

## Performance (design-adjacent)

- Fonts: self-host, `woff2`, subset, `font-display: optional` (or `swap`) + preload critical
  faces; use metric-override fallbacks (`size-adjust`/`ascent-override`) or `next/font` to kill
  font-CLS. Font swaps count toward CLS (good ≤0.1).
- Images: correct sizes/formats, `loading="lazy"` below fold, explicit dimensions to avoid CLS,
  `aspect-ratio` to reserve space.
- Ship less CSS/JS; lazy-load animation libs (`41`). Core Web Vitals are a design constraint,
  gated in `82`.

## Delivery checklist

- [ ] Tailwind v4 CSS-first tokens (OKLCH), container queries, `data-slot`
- [ ] Native dialog/popover/anchor/view-transitions where they fit, with `@supports` fallbacks
- [ ] Cascade layers; logical properties; `gap` not `space-*`
- [ ] Font + image CLS handled; CWV budgeted (`82`)
